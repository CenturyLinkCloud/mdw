MDW Axis-based Webservice Adapter Activity

This adpater uses Axis 1.4 for completely dynamic invocations. It supports both doc/lit and rpc/encoded services.

Configuring Adapter Activity

  • WSDL url
    • Specify url of WSDL. The value of this attribute can be either static or property driven
  • Operation Nm
    • Name of the operation in WSDL. The value of this attribute can be either static or property driven
  • Log Response
    • Boolean value of true or false for logging the response into log files
  • Request Params
    • This specifies list of request parameters to the request. The value(s) could be static for e.g. JohnDoe or any process variables for e.g. $zipCode or $username[current_user]
    • For dynamic binding of process variable values to service input, adapter uses groovy parser.

Handling Response from service

Typically, a developer or analyst would write a groovy script which gets executed after service response comes back. The adapter analyze response from service and organizes into "response" variable of type java.util.Hashmap. The key in map represents the name of response variable(s) defined in WSDL operation. This variable is binded to groovy script and available for developers to manipulate.

Sample groovy code: VPN_ID= response.get("return")

The variable response is NOT a process variable.

In groovy script a user could write code to manipulate return code of transition. A new feature available as of MDW Release 4.2.12 - This adapter can set returncode to "ABORT:", "CANCEL:","CORRECTION:" which could be use to raise those three events
