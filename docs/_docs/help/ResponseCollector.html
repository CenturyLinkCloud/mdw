Response Collector Activity

The Response Collector activity handles consolidation of multiple service invocations into a single Status Response.

Consolidator Implementation

Responses are combined through a Consolidator. The default (embedded in the activity implementor class) is to set the code to the maximum HTTP status code among the responses, and to populate the JSON payload with a map of accumulated StatusResponses.

This behavior can be overridden by entering your a value for the Consolidator attribute. This should be the fully-qualified class name of your custom Consolidator implementation (in Java or Kotlin).
